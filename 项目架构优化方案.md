## 2. 优化方案（按优先级）

### P0：安全与正确性修复

#### **P0-1：启用 SQLite 外键约束和 WAL 模式**

- **位置**：`src-tauri/src/db.rs`，`init_db` 函数开头
- **修改**：在 `Connection::open` 之后立即执行：
  ```sql
  PRAGMA foreign_keys = ON;
  PRAGMA journal_mode = WAL;
  ```
- **风险**：WAL 模式会生成额外的 `.db-wal` 和 `.db-shm` 文件，需确保不误删

#### **P0-2：处理 Mutex 中毒**

- **位置**：`src-tauri/src/commands.rs`，16 处 `state.0.lock().unwrap()`
- **修改**：全部替换为：
  ```rust
  let conn = state.0.lock().map_err(|e| format!("数据库锁错误: {}", e))?;
  ```
- **风险**：无，纯错误处理增强

#### **P0-3：设置 CSP 策略**

- **位置**：`src-tauri/tauri.conf.json` 第 28-30 行
- **修改**：
  ```json
  "csp": "default-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self' https://api.openai.com https://*.openai.com; font-src 'self' data:; img-src 'self' data: blob:"
  ```
- **注意**：如果 AI 使用了非 OpenAI 的 API 地址，需在 `connect-src` 中添加对应域名

#### **P0-4：添加数据库索引**

- **位置**：`src-tauri/src/db.rs`，在 `init_db` 末尾统一创建
- **修改**：添加以下索引：
  ```sql
  CREATE INDEX IF NOT EXISTS idx_tasks_list_id ON tasks(list_id);
  CREATE INDEX IF NOT EXISTS idx_tasks_parent_id ON tasks(parent_id);
  CREATE INDEX IF NOT EXISTS idx_tasks_completed ON tasks(completed);
  CREATE INDEX IF NOT EXISTS idx_tasks_archived ON tasks(archived);
  CREATE INDEX IF NOT EXISTS idx_tasks_pinned ON tasks(pinned);
  CREATE INDEX IF NOT EXISTS idx_tasks_sort_order ON tasks(pinned DESC, sort_order ASC, created_at DESC);
  CREATE INDEX IF NOT EXISTS idx_task_tags_task_id ON task_tags(task_id);
  CREATE INDEX IF NOT EXISTS idx_task_tags_tag_id ON task_tags(tag_id);
  ```
- **风险**：索引会增加磁盘空间，建索引期间会短暂锁表（数据量不大时无感知）

       let exists = stmt.query_map([], |row| row.get::<_, String>(1))?
