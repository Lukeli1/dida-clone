# 滴答清单右键菜单功能改进文档

> 基于滴答清单原版截图分析，对比当前复刻版本实现状态，制定改进计划

---

## 一、功能对照总览

| 序号 | 原版功能 | 分类 | 当前复刻状态 | 改进优先级 |
|------|----------|------|-------------|-----------|

## 二、功能逐项分析

### 1. 日期（子菜单）

#### 原版截图表现
- 标题"日期"下四个快捷按钮：☀️（今天）、🌅（明天）、7+（7天后）、📅（自定义）、✕（清除截止日期）

#### 当前复刻实现
- ❌ 右键菜单中**完全没有日期快捷操作**
- ✅ 但已有底层能力：`due_date` 字段、`updateTask` API、日历拖拽设置日期
- ✅ 详情页（TaskDetail）中已有完整的日期选择器

#### 改进方案

- 在右键菜单中增加"日期"分组标题 + 快捷按钮组，包含今天、明天、7天后、自定义（弹出日期选择器）、清除五个按钮
- 新增 `onSetDate` 回调接口，支持 `'today' | 'tomorrow' | '7days' | 'custom'` 类型参数
- 新增 `onClearDate` 回调接口，用于清除截止日期
- 日期计算逻辑：今天返回当前日期、明天加 1 天、7天后加 7 天，自定义通过弹窗选择
- **后端/数据库**：无需改动，`due_date` 字段已存在
- **实现优先级**：高 — 与优先级同为右键菜单核心快捷操作

---

### 2. 优先级（子菜单）

#### 原版截图表现
- 标题"优先级"下四个快捷按钮：🔴（高/红旗）、🟡（中/黄旗）、🔵（低/蓝旗）、⬜（无/灰旗）

#### 当前复刻实现
- ⚠️ **批量操作工具栏**中已有机能优先级下拉（高/中/低/无）
- ✅ `priority` 字段已存在（0=无，1=高，2=中，3=低）
- ❌ **右键菜单中无优先级快捷操作**

#### 改进方案

- 在右键菜单中增加"优先级"分组标题 + 四个快捷按钮（高/中/低/无），每个按钮使用对应颜色的旗帜图标
- 新增 `onSetPriority` 回调接口，接收 `number` 类型参数（0/1/2/3）
- **后端/数据库**：无需改动，`priority` 字段已存在，`updateTask` API 已支持
- **实现优先级**：高 — 与日期同为右键菜单核心快捷操作

---

### 3. 置顶

#### 原版截图表现
- "置顶"文字 + ↑ 图标，点击后任务固定在列表顶部

#### 当前复刻实现
- ❌ **完全缺失**
- 原因：`tasks` 表中无 `pinned` 字段
- 现有排序机制基于 `sort_order`（拖拽排序），但没有独立的"置顶"概念

#### 改进方案

**数据库改动**：
- 在 `tasks` 表中新增 `pinned` 列（INTEGER DEFAULT 0），需通过 `PRAGMA table_info` 检测已有数据库是否含该列，缺失则用 `ALTER TABLE ADD COLUMN` 兼容
- Rust `Task` struct 中新增 `pinned: bool` 字段
- SQL 查询中读取 `pinned` 字段

**前端改动**：
- TypeScript `Task` 接口新增 `pinned?: boolean` 字段
- `UpdateTaskRequest` 新增 `pinned?: boolean`
- `update_task` Tauri command 支持更新 `pinned` 字段
- 右键菜单中添加"置顶"按钮，根据 `task.pinned` 状态显示"置顶"或"取消置顶"，置顶状态时用橙色高亮
- 新增 `onTogglePin` 回调接口
- 排序逻辑中优先排 pinned 任务到顶部（置顶 > 未完成 > 已完成 > sort_order）
- 置顶任务在列表项左侧增加置顶图标标识

**实现优先级**：高 — 置顶是高频使用的任务管理功能

---

### 4. 标签（子菜单）

#### 原版截图表现
- "标签"文字 + 🏷️ 图标，子菜单列出所有已有标签（可勾选），底部有"新建标签"选项

#### 当前复刻实现
- ⚠️ **标签体系已基本完整**：
  - ✅ 标签 CRUD（创建/删除/查询）已实现
  - ✅ 任务-标签关联（`addTagToTask` / `removeTagFromTask`）已实现
  - ✅ 详情页（TaskDetail）中有标签添加/移除界面
  - ❌ **右键菜单中无标签快捷操作**

#### 改进方案

- 在右键菜单中增加"标签"子菜单（hover 展开或点击展开）
- 子菜单中列出所有已有标签，每个标签前有勾选框，显示当前任务是否已关联该标签
- 子菜单底部有"新建标签"入口，复用侧边栏的标签创建逻辑
- 新增 `onToggleTag` 回调接口，接收 `(taskId, tagId)` 参数
- 新增 `onCreateNewTag` 回调接口
- **后端/数据库**：无需改动，标签相关 API 已完整
- **实现优先级**：中 — 标签功能已在详情页可用，右键快捷是体验优化

---

### 5. 创建副本

#### 原版截图表现
- "创建副本"文字 + 📋 图标，点击后创建一份完全相同的任务副本

#### 当前复刻实现
- ❌ **完全缺失**
- ✅ 有底层能力：`createTask` API、`Task` 类型完整
- ✅ 重复任务完成（`completeTask`）时已有复制标签的逻辑，可作为参考

#### 改进方案

**后端改动**：
- 新增 `duplicate_task` Tauri command，接收任务 ID，查询原任务所有字段后插入副本（不复制 completed 状态、id、created_at、updated_at）
- 副本需复制标签关联（从 `task_tags` 表读取原任务的所有 tag_id，插入到新任务）
- 返回新创建的 Task 对象

**前端改动**：
- 右键菜单中添加"创建副本"按钮
- API 层新增 `duplicateTask` 方法，同时支持 Tauri 和 mock 模式
- 新增 `onDuplicate` 回调接口
- Mock 模式下：复制源任务的所有字段（除 id/completed/sort_order/created_at/updated_at），生成新任务并插入列表头部

**实现优先级**：中 — 实用但非最高频功能

---

### 6. 删除

#### 原版截图表现
- "删除"文字 + 🗑️ 图标，红色高亮

#### 当前复刻实现
- ✅ **已完整实现**：右键菜单中有"删除任务"按钮，红色样式
- ✅ `deleteTask` API + `delete_task` Tauri command 完整
- ✅ 包含子任务级联删除、标签关联清理
- ⚠️ 可优化：缺少删除确认弹窗（原版有）

#### 改进方案

当前实现已基本满足需求，唯一可优化项：

- 增加删除确认弹窗，避免误操作
- 可使用自定义 Toast 确认而非浏览器原生 `confirm`

**实现优先级**：低 — 当前已可用

## 四、技术实现路线图

### Phase 1：右键菜单核心快捷操作（1-2天）

1. **日期快捷设置** — 在右键菜单中添加日期按钮组，通过回调调用 `updateTask` API 设置 `due_date`
2. **优先级快捷设置** — 在右键菜单中添加优先级按钮组，通过回调调用 `updateTask` API 设置 `priority`

### Phase 2：数据模型扩展（1天）

3. **置顶功能** — 数据库添加 `pinned` 列，TypeScript types 添加 `pinned` 字段，右键菜单添加置顶按钮，排序逻辑优先排 pinned

### Phase 3：标签子菜单（1天）

4. **标签子菜单** — 右键菜单中添加标签子菜单，展示已有标签勾选列表 + 新建标签入口

### Phase 4：任务副本（1天）

5. **创建副本** — 后端新增 `duplicate_task` command，前端右键菜单添加副本按钮

### Phase 5：清理与优化（半天）

6. **删除确认** — 增加删除确认弹窗
7. **UI 一致性** — 确保右键菜单样式与原版截图一致（圆角、阴影、分组间距、图标风格）

## 七、注意事项

1. **数据库兼容**：所有新增字段（如 `pinned`）必须通过 `PRAGMA table_info` 检测 + `ALTER TABLE ADD COLUMN` 方式兼容已有数据库
2. **排序逻辑**：置顶任务的排序需在 `App.tsx` 的 `filteredTasks` sort 函数中优先处理，不影响现有 `sort_order` 拖拽排序
3. **Mock 模式**：浏览器预览模式下所有新功能必须同步实现 mock 逻辑，保证离线可用
4. **子任务处理**：置顶、优先级、日期等操作目前仅作用于顶层任务，子任务不变（与现有一致）
5. **标签子菜单交互**：建议使用 hover 展开子菜单（如原版），而非 click 展开，减少操作步骤
