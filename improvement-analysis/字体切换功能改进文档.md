# 滴答清单字体切换功能改进文档

> 在设置界面增加显示字体切换功能，允许用户自定义应用全局字体，提升阅读体验

---

## 一、功能概述

### 1.1 目标

在设置界面「通用」分区新增字体切换选项，提供以下能力：
- **预设字体列表**：提供 5~8 种中英文常用字体供一键切换
- **自定义字体**：支持用户输入任意系统已安装的字体名称
- **即时生效**：切换字体后全界面即时更新，无需重启
- **持久化**：设置保存至 `localStorage`，下次启动自动恢复

### 1.2 当前状态

| 项目 | 现状 |
|------|------|
| 全局字体 | 硬编码在 `index.css` 的 `body` 上（system-ui 回退栈） |
| Tailwind 字体族 | `tailwind.config.js` 中 `fontFamily.sans` 与 body 一致 |
| 字体设置入口 | ❌ 不存在 |
| 字体持久化 | ❌ 不存在 |
| 组件级字体覆盖 | ✅ 无（均为继承全局字体，改造干净） |

---

## 二、技术方案设计

### 2.1 核心架构

采用 **CSS 自定义属性（CSS Variables） + `localStorage` 持久化** 方案，与现有主题切换机制完全一致：

```
用户设置 SettingsView → localStorage('appFont') → CSS 变量
                                                    ↓
                                              document.documentElement.style.setProperty()
                                                    ↓
                                              body / 全局组件继承新字体
```

### 2.2 字体预设列表

| 显示名 | 字体值 | 适用场景 |
|--------|--------|----------|
| 系统默认 | `system-ui, -apple-system, sans-serif` | 默认回退 |
| 苹方 (PingFang SC) | `"PingFang SC", system-ui, sans-serif` | macOS/iOS 首选 |
| 微软雅黑 | `"Microsoft YaHei", system-ui, sans-serif` | Windows 首选 |
| 思源黑体 (Noto Sans SC) | `"Noto Sans SC", "Source Han Sans SC", system-ui, sans-serif` | 跨平台通用 |
| 霞鹜文楷 | `"LXGW WenKai", "霞鹜文楷", system-ui, serif` | 楷体风格 |
| 思源宋体 (Noto Serif SC) | `"Noto Serif SC", "Source Han Serif SC", Georgia, serif` | 阅读/长文 |
| JetBrains Mono | `"JetBrains Mono", "Fira Code", monospace` | 代码/极客风格 |
| 自定义 | 用户输入 | 任意系统字体 |

> 预设字体在 `tailwind.config.js` 中以扩展 `fontFamily` 的形式注册，便于在组件中使用 `font-*` 工具类时保持一致。

### 2.3 存储设计

**键名**：`appFont`

**数据结构**：

```typescript
// 预设选中 -> 存 preset 标识
// 自定义输入 -> 存完整 font-family 字符串
type AppFontSetting = 
  | { type: 'preset'; key: string }      // 预设字体标识
  | { type: 'custom'; value: string }     // 自定义字体值
```

**存储示例**：
```
localStorage.setItem('appFont', JSON.stringify({ type: 'preset', key: 'pingfang' }))
localStorage.setItem('appFont', JSON.stringify({ type: 'custom', value: '"MyCustomFont", sans-serif' }))
```

### 2.4 字体应用机制

```typescript
// 应用字体到全局
function applyFont(setting: AppFontSetting) {
  const fontMap: Record<string, string> = {
    system:    'system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans", sans-serif',
    pingfang:  '"PingFang SC", "Microsoft YaHei", system-ui, -apple-system, sans-serif',
    msyahei:   '"Microsoft YaHei", "PingFang SC", system-ui, -apple-system, sans-serif',
    noto:      '"Noto Sans SC", "Source Han Sans SC", system-ui, -apple-system, sans-serif',
    lxgw:      '"LXGW WenKai", "霞鹜文楷", system-ui, -apple-system, serif',
    notoserif: '"Noto Serif SC", "Source Han Serif SC", Georgia, "Times New Roman", serif',
    jetbrains: '"JetBrains Mono", "Fira Code", "Consolas", monospace',
  }

  const fontValue = setting.type === 'preset' 
    ? (fontMap[setting.key] || fontMap.system)
    : setting.value

  // 用 CSS 变量挂载到根元素
  document.documentElement.style.setProperty('--app-font-family', fontValue)
}
```

## 四、实现步骤

### Step 1 — Tailwind 配置扩展（`tailwind.config.js`）

在 `theme.extend.fontFamily` 中注册全部预设字体：

```js
fontFamily: {
  sans: ['system-ui', '-apple-system', ...],
  pingfang: ['"PingFang SC"', '"Microsoft YaHei"', 'system-ui', ...],
  msyahei: ['"Microsoft YaHei"', '"PingFang SC"', 'system-ui', ...],
  noto: ['"Noto Sans SC"', '"Source Han Sans SC"', 'system-ui', ...],
  lxgw: ['"LXGW WenKai"', '"霞鹜文楷"', 'system-ui', ...],
  notoserif: ['"Noto Serif SC"', '"Source Han Serif SC"', 'Georgia', ...],
  jetbrains: ['"JetBrains Mono"', '"Fira Code"', ...],
}
```

### Step 2 — 新增字体工具函数（`src/utils/font.ts`）

创建 `src/utils/font.ts`，包含：

- `PRESET_FONTS` 常量（预设列表）
- `getFontSetting()` — 从 localStorage 读取
- `saveFontSetting(setting)` — 写入 localStorage
- `applyFont(setting)` — 将字体应用到 `document.documentElement` 的 CSS 变量
- `getFontCSSValue(key)` — 根据预设 key 返回完整 font-family 字符串
- `getFontDisplayName(key)` — 根据预设 key 返回中文显示名

### Step 3 — 修改 `index.css`

在 `body` 的 `font-family` 中增加 CSS 变量回退：

```css
body {
  font-family: var(--app-font-family, system-ui, -apple-system, ...);
}
```

这样：变量存在时用变量值，不存在（未设置）时回退到默认字体。

### Step 4 — 修改 `SettingsView.tsx`

在「通用」section 的最上方新增「显示字体」设置行：

```tsx
// 状态
const [fontSetting, setFontSetting] = useState<AppFontSetting>(() => getFontSetting())
const [customFontInput, setCustomFontInput] = useState('')

// 选择预设
function handleFontPresetSelect(key: string) {
  const setting = { type: 'preset' as const, key }
  setFontSetting(setting)
  saveFontSetting(setting)
  applyFont(setting)
}

// 确认自定义字体
function handleCustomFontConfirm() {
  if (!customFontInput.trim()) return
  const setting = { type: 'custom' as const, value: customFontInput.trim() }
  setFontSetting(setting)
  saveFontSetting(setting)
  applyFont(setting)
}
```

UI 渲染：

```tsx
<div className="flex items-center justify-between px-4 py-3.5">
  <div>
    <p className="text-sm font-medium text-gray-900">显示字体</p>
    <p className="text-xs text-gray-500 mt-0.5">选择应用的全局显示字体</p>
  </div>
  <select
    value={fontSetting.type === 'preset' ? fontSetting.key : 'custom'}
    onChange={(e) => {
      if (e.target.value === 'custom') {
        // 展开自定义输入
      } else {
        handleFontPresetSelect(e.target.value)
      }
    }}
    className="px-3 py-1.5 text-sm border border-gray-200 rounded-lg bg-white"
  >
    {PRESET_FONTS.map(f => (
      <option key={f.key} value={f.key}>{f.displayName}</option>
    ))}
    <option value="custom">自定义...</option>
  </select>
</div>
{fontSetting.type === 'custom' && (
  <div className="flex items-center gap-2 px-4 pb-3.5">
    <input
      type="text"
      value={customFontInput}
      onChange={(e) => setCustomFontInput(e.target.value)}
      placeholder="输入字体名称，如 'Noto Serif SC'"
      className="flex-1 px-3 py-1.5 text-sm border border-gray-200 rounded-lg"
    />
    <button onClick={handleCustomFontConfirm} className="px-3 py-1.5 text-sm bg-blue-500 text-white rounded-lg">
      应用
    </button>
  </div>
)}
```

### Step 5 — 修改 `App.tsx`

在启动时的 `useEffect` 中添加字体恢复逻辑（与主题恢复并列）：

```tsx
// 启动时恢复字体设置
const savedFont = getFontSetting()
if (savedFont) applyFont(savedFont)
```

### Step 6 — 重新构建

```bash
npm run build
```

## 六、注意事项 / 边界情况

| 问题 | 处理方案 |
|------|----------|
| **字体不存在** | 浏览器自动回退到 CSS font-family 栈中下一个可用字体，无需额外处理 |
| **自定义字体输入错误** | 即使输入无效字体名，浏览器也会静默回退，不出错 |
| **中文字体兼容性** | 预设均为跨平台中文字体，但 LXGW WenKai / Noto Serif SC 可能需要用户自行安装，提供提示说明 |
| **与 Tailwind font-* 类冲突** | 组件中若使用 `font-sans` / `font-mono` 等类，会覆盖全局字体。需检查是否有关键组件硬编码了字体类 |
| **性能影响** | 切换字体本质是修改 CSS 变量，仅触发一次重排，无性能问题 |
| **Tauri 环境** | Tauri v2 使用 WebView2 (Windows) / WKWebView (macOS)，系统字体可用性与浏览器一致 |
| **字体加载延迟**（Web 字体） | 当前方案仅使用系统已安装字体，不涉及网络加载；若未来引入 Google Fonts，需处理 `document.fonts.ready` |
| **与其他设置的兼容** | 字体变量与深色模式 / 主题切换互不干扰，独立层叠 |
