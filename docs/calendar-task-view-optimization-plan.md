# 日历任务视图显示优化方案

> 创建时间：2026-07-08
> 适用项目：滴答清单复刻
> 范围：日历模块（月视图 / 周视图 / 日视图）及任务显示 UI
> 约定：后续每次开启或完成任一阶段任务时，都必须先对照本文件确认阶段目标、验收标准与未完成项。

## 1. 已确认的设计决策

1. **月视图已完成任务默认折叠**
   - 已完成任务不占用日期格主显示位。
   - 日期格只显示已完成摘要，例如 `✓ 3 已完成`。

2. **月视图更多任务使用原地浮层**
   - 点击 `+N` 后，在日期格附近打开浮层。
   - 不再使用全屏遮罩 + 居中弹窗作为主要展示方式。
   - 点击外部关闭，点击任务后关闭浮层并打开任务详情。

3. **任务颜色策略**
   - 清单色为主。
   - 优先级作为辅助表达，例如小旗、红点、左侧强调等。
   - 不用优先级颜色完全替代清单颜色，避免任务归属感丢失。

4. **周/日视图后续增加顶部“全天 / 跨天任务区”**
   - 作为跨天任务和全天任务的专用展示区域。
   - 不与时间轴内普通任务块混在一起。

## 2. 总体优化目标

1. **看得更多**：同样空间下展示更多有效任务。
2. **看得更清楚**：优先级、时间、完成状态、重复/提醒等关键信息一眼可见。
3. **不遮挡**：周/日视图重叠任务要并排或分组显示。
4. **不丢上下文**：更多任务、悬浮预览、详情入口尽量贴近日历格。
5. **可扩展**：为后续工作日视图、全天任务、过滤器、显示密度设置打基础。

## 3. 当前关键问题

### 3.1 月视图空间利用率偏低

当前 `MonthView.tsx` 使用固定常量：

```ts
const MAX_MONTH_VISIBLE_TASKS = 2
```

问题：

- 大屏幕下日期格有空白，但仍只显示 2 条。
- 任务多的日期需要频繁点 `+N`。
- 已完成任务虽然排在后面，但仍占用有限显示区域。
- `+N` 弹窗是居中的全屏遮罩，视觉上下文会断掉。

### 3.2 周视图 / 日视图同时间段任务会互相覆盖

当前周/日任务块按 `top + height + absolute left-1 right-1` 渲染，没有计算重叠列。重叠时间段任务可能互相遮挡。

### 3.3 跨天任务没有完整显示

任务模型已有 `due_date` / `end_date`，但月视图主要按 `due_date` 分组。跨天任务目前不具备完整连续展示。

### 3.4 任务条信息密度不足

当前任务条主要展示复选框、标题、时间；提醒、重复、备注、子任务数量、高优先级等信息需要进入详情才能确认。

### 3.5 月/周/日任务组件不统一

当前月/周视图使用 `TaskBar.tsx`，日视图使用 `DayViewTask.tsx`。颜色、内容、交互、状态表达分散，后续维护成本较高。

## 4. 阶段路线图

## 阶段 1：月视图任务显示优化

### 目标

- 未完成任务优先显示。
- 已完成任务默认折叠。
- 日期格根据空间显示更多任务。
- `+N` 更多任务改成原地浮层。
- 任务条展示更多关键信息，但不拥挤。

### 主要改动

1. **任务分组逻辑调整**
   - 按日期分组后，拆成：
     - `active`：未完成任务
     - `completed`：已完成任务
   - 日期格主区域优先展示未完成任务。

2. **动态显示任务数量**
   - 替换固定 `MAX_MONTH_VISIBLE_TASKS = 2`。
   - 第一版可采用稳定估算策略：根据日期格高度估算可显示数量。
   - 建议范围：最少 2 条，最多 5 条。

3. **更多任务原地浮层**
   - 点击 `+N` 打开日期格附近浮层。
   - 浮层展示当天未完成任务与已完成任务。
   - 点击外部关闭。
   - 点击任务后关闭浮层并打开详情。
   - 浮层高度限制，内部滚动。

4. **已完成任务折叠入口**
   - 日期格显示摘要：`✓ N 已完成`。
   - 已完成列表默认不直接占用日期格主显示位。
   - 完整已完成列表在原地浮层中查看。

5. **任务条信息增强**
   - 保留复选框、标题、时间。
   - 增加轻量标识：
     - 高优先级
     - 提醒
     - 重复
     - 子任务数量
     - 备注
   - 避免过度拥挤，必要时只显示最重要的 2-3 个标识。

### 涉及文件

- `src/components/calendar/MonthView.tsx`
- `src/components/calendar/shared/TaskBar.tsx`
- 可新增：
  - `src/components/calendar/MonthMorePopover.tsx`
  - `src/components/calendar/shared/CalendarTaskBadges.tsx`

### 阶段 1 验收标准

- [x] 日期格优先显示未完成任务。
- [x] 已完成任务默认不占主显示位。
- [x] 日期格显示数量比固定 2 条更灵活。
- [x] 点击 `+N` 是原地浮层，不是全屏居中弹窗。
- [x] 浮层中可以看到当天所有未完成和已完成任务。
- [x] 任务条能显示提醒 / 重复 / 备注 / 子任务 / 高优先级等轻量标识。
- [x] 原有拖拽改期、点击详情、勾选完成、快速添加不被破坏。

## 阶段 2：周视图 / 日视图重叠任务布局

### 目标

解决同时间段任务互相遮挡的问题。

### 主要改动

1. 新增纯函数，例如 `layoutTimedTasks(tasks)`。
2. 输出每个任务的：

```ts
{
  task,
  top,
  height,
  leftPercent,
  widthPercent
}
```

3. 周视图和日视图共用该布局函数。
4. 重叠任务并排显示。

### 建议新增文件

- `src/utils/calendarTaskLayout.ts`

### 阶段 2 验收标准

- [ ] 完全重叠任务不再互相遮挡。
- [ ] 部分重叠任务可并排显示。
- [ ] 周视图和日视图使用同一布局算法。
- [ ] 拖拽、resize、点击详情、勾选完成不被破坏。
- [ ] 布局算法具备单元测试。

## 阶段 3：统一日历任务条组件

### 目标

统一月/周/日任务条的视觉与交互。

### 建议方向

将 `TaskBar.tsx` 与 `DayViewTask.tsx` 逐步收敛为：

```tsx
<CalendarTaskBlock />
```

统一处理：

- 颜色
- 标题
- 时间
- 复选框
- 提醒 / 重复 / 备注 / 子任务图标
- hover
- dragging
- resizing
- completed 状态

### 阶段 3 验收标准

- [ ] 月/周/日视图任务条视觉一致。
- [ ] 公共颜色、图标、完成态逻辑不重复实现。
- [ ] 原有交互不回退。

## 阶段 4：全天 / 跨天任务区

### 目标

完整支持全天任务与跨天任务显示。

### 建议方向

引入中间数据结构：

```ts
interface CalendarOccurrence {
  task: Task
  dateKey: string
  start: Date
  end?: Date
  isAllDay: boolean
  isMultiDay: boolean
  segment: 'start' | 'middle' | 'end' | 'single'
}
```

周/日视图顶部增加全天/跨天区域；月视图支持跨天连续条带。

### 阶段 4 验收标准

- [ ] 跨天任务在月视图中具备连续感。
- [ ] 周/日视图顶部有全天/跨天任务区。
- [ ] 点击任意跨天段都能打开同一个任务详情。
- [ ] 普通时间轴任务显示不受影响。
