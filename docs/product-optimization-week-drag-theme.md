# 日历跨日规划与完整主题体验优化 PRD

> 状态：阶段 1（v1.44.0）和阶段 2（v1.45.0）均已完成并发布
> 产品方向：个人高频任务管理，优先减少日历规划步骤并提升长期使用舒适度
> 范围：周/月视图跨日拖拽创建、周视图边缘自动滚动、完整主题包与有限个性化

## 1. 背景与痛点

### 1.1 周视图跨日创建

当前周视图已支持单日时间轴拖选创建，但选择被限制在单个日期列。创建跨日定时任务时，用户需要先创建任务，再进入详情分别修改开始日期、开始时间、结束日期和结束时间，步骤多且容易填错。

### 1.2 主题系统

当前已支持浅色、深色、跟随系统、6 套配色预设和自定义强调色，但多数预设只改变强调色，背景、表面、边框、文字层级和深色适配仍不完整。

## 2. 已确认产品决策

1. 跨日任务创建后继续显示在周视图顶部的全天/跨天任务区，并展示精确起止日期和时间，不在时间轴中绘制覆盖多个整天的大块任务。
2. 支持正向和反向拖拽，松开后自动按时间先后归一化。
3. 主题系统升级为约 10 套完整主题，每套具有独立浅色和深色变量；保留自定义强调色并增加圆角风格。
4. 不做壁纸、动态背景、主题商城、在线分享或完全自由的 CSS 变量编辑器。

## 3. 阶段 1：v1.44.0 日历跨日拖拽创建

### 3.1 用户流程

1. 用户在周视图时间轴空白区域或月视图日期格空白区域按下主 Pointer。
2. 拖到同日或另一日期的目标时间，时间按 15 分钟吸附。
3. 拖动中按日期分段显示半透明预览，并显示完整起止时间和持续时长。
4. 松开后打开创建弹窗，预填开始/结束日期时间。
5. 用户可编辑标题、日期时间、备注、优先级和清单后创建任务。

### 3.2 交互规则

- 支持同日、跨日、向左、向右、向上和向下拖动。
- 反向拖动自动归一化为较早时间到较晚时间。
- 第一版仅在当前可见的 7 天中拖选；跨周范围可在创建弹窗中编辑。
- 不足 15 分钟的短按继续按现有逻辑创建默认 1 小时时段。
- 交互优先级：任务 resize > 移动任务 > 点击任务 > 空白时间轴拖选创建。
- Pointer 离开原始日期列、在网格外松开或触发 pointercancel 时不得残留选区。

### 3.3 预览规则

- 起始日：开始时间到 24:00。
- 中间日：00:00 到 24:00。
- 结束日：00:00 到结束时间。
- 同日：开始时间到结束时间。
- 浮动提示展示“周一 09:00 → 周三 17:30”和持续时长。
- 使用独立语义变量适配浅色和深色主题。

### 3.4 创建弹窗

字段：标题、开始日期/时间、结束日期/时间、备注、优先级、清单。

校验：

- 标题不能为空。
- 结束时间必须严格晚于开始时间。
- 校验失败或创建 API 失败时不得关闭弹窗，并保留用户输入。
- Esc 取消；标题输入框 Enter 创建；textarea Enter 仅换行。
- 弹窗位置需限制在可视窗口范围内。

### 3.5 创建后展示

跨日任务继续进入顶部全天/跨天任务区，任务条显示标题及精确起止时间。普通全天任务、跨周分段、完成状态、清单颜色、点击详情和拖动行为不得回退。

### 3.6 响应式

- 周视图宽窗口显示完整 7 天，拖到上下或左右边缘时自动滚动以继续选择不可见时间。
- 窄窗口为时间网格设置合理最小宽度并允许横向滚动。
- 坐标换算必须在滚动后仍准确。
- 第一版以桌面鼠标/Pointer 为主，为触控保留 Pointer Events 基础，不承诺完整长按拖选。

### 3.7 验收标准

- [x] 同日拖选行为不回退。
- [x] 周一 09:00 拖到周三 17:30 可正确创建跨日任务。
- [x] 周视图拖到可视区域边缘时自动滚动，可选择 23:00 等当前不可见时间。
- [x] 月视图可跨日期格拖选范围，并预填起止日期时间。
- [x] 反向跨日拖动可自动归一化。
- [x] 实时预览分段、边界、文字在浅色和深色下清晰。
- [x] 创建弹窗预填并允许编辑起止日期时间。
- [x] due_date/end_date 使用本地日期构造后转 ISO，不发生日期偏移。
- [x] 跨日顶部任务条显示精确起止时间。

## 4. 阶段 2：v1.45.0 完整主题系统

### 4.1 目标

将当前强调色预设升级为完整主题包。每套主题分别定义浅色和深色的背景、表面、边框、文字、强调色、日历选择态和阴影。

### 4.2 首批主题

经典蓝、森林绿、薰衣草紫、暖阳橙、樱花粉、莫兰迪、海洋青、石墨灰、纸张米白、午夜蓝。

### 4.3 个性化范围

- 浅色、深色、跟随系统。
- 完整主题预览卡。
- 自定义强调色。
- 圆角风格：紧凑、标准、柔和。
- 界面密度：紧凑、舒适、宽松。
- 恢复主题默认值。

### 4.4 安全与无障碍

- 普通文字对比度尽量达到 4.5:1，大文字达到 3:1。
- 自定义强调色自动选择可读按钮文字。
- 错误、成功、警告和优先级颜色不完全跟随强调色。
- 无效主题配置自动回退经典蓝。

### 4.5 验收标准

- [x] 提供经典蓝、森林绿、薰衣草紫、暖阳橙、樱花粉、莫兰迪、海洋青、石墨灰、纸张米白、午夜蓝共 10 套主题。
- [x] 每套主题具有独立浅色和深色的背景、表面、边框、文字、强调色、日历选择态和阴影变量。
- [x] 浅色、深色和跟随系统切换后立即应用，系统模式变化时无需保持设置页打开。
- [x] 设置页以完整界面缩略图预览主题，而不是只显示强调色圆点。
- [x] 自定义强调色保留主题背景体系，并自动选择达到 4.5:1 的按钮文字颜色。
- [x] 圆角支持利落、标准、柔和三档，并应用到常用 Tailwind 圆角工具类。
- [x] 界面密度支持紧凑、舒适、宽松三档，并影响侧边栏与高频任务行。
- [x] 主题、强调色、圆角和密度持久化，兼容旧 localStorage key；无效配置回退经典蓝。
- [x] 错误、成功、警告和优先级语义色不随自定义强调色整体替换。
- [x] 主题结构、对比度、持久化、设置面板和窄窗口具有自动化测试。
- [x] 在当前 Tauri 桌面版人工复验 10 套主题的主要视图、弹窗和深浅色视觉效果。

## 5. 版本建议

- v1.44.0：周/月视图跨日拖拽创建任务与周视图边缘自动滚动。
- v1.45.0：完整主题系统。

两个阶段独立验证和发布，避免日期交互与全局视觉变量同时变更。
